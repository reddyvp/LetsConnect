## <span style="color:#E50914">в—ј KEY SCENES</span>

### `[SCENE 1]` вҖ” The Swipe
- Drag cards **left** вҶ’ reject with вң• indicator
- Drag cards **right** вҶ’ like with вҷҘ indicator
- **30% chance** of mutual match on right swipe
- Undo last swipe with the revert button
- Reset deck when all profiles are viewed

### `[SCENE 2]` вҖ” The Match
- Full-screen match celebration modal fires
- Both profiles displayed side by side
- CTA to message immediately or keep swiping

### `[SCENE 3]` вҖ” The Conversation
- Real-time style chat interface
- User messages: **gradient styled** (right side)
- Match messages: **white bubble** (left side)
- Auto-scrolls to latest message
- Send button disabled on empty input

- Name, age, occupation

## <span style="color:#E50914">в—ј CLAUDE DIRECTIVES</span>

When working on this codebase:

1. **Design system first** вҖ” Always use Tailwind utilities. No inline styles unless Motion requires it.
2. **Mobile-first** вҖ” This is a phone-native experience. Design for `375px` width upward.
3. **Mock data** вҖ” All data lives in `src/app/data/mockData.ts`. No backend, no API calls.
4. **Animations** вҖ” Use the `motion` library for all drag and transition effects. Keep them snappy (<300ms).
5. **Component scope** вҖ” Each page is self-contained. Don't introduce cross-page state without good reason.
6. **TypeScript strict** вҖ” All new code must use the types defined in `src/app/types.ts`.
7. **No new dependencies** вҖ” The stack is mature. Solve problems with what's already installed.
